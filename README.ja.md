# Bookmark Organizer

[English](README.md) | [中文](README.zh-CN.md) | [Español](README.es.md) | **日本語** | [Deutsch](README.de.md)

ブラウザのブックマークを放置して蓄積させるのは苦痛です。何百ものブックマークが蓄積された後、何かを見つけるには検索が必要になり、お気に入りバーは役に立たなくなります。この拡張機能は、その問題を解決するために設計されています。

Bookmark Organizer は、散らかったブックマークを自動的にカテゴリ分けし、重複を見つけ、変更前にバックアップを作成するブラウザ拡張機能です。何か問題が発生した場合でも、ワンクリックで復元できます。すべてのデータ処理はデバイス上でローカルに行われ、サーバーに何もアップロードされません。

---

## 機能

### 1. スマート分類
すべてのブックマークをスキャンし、タイトル、URL、ドメインに基づいて自動的にカテゴリを提案します。例えば、「github」や「stackoverflow」を含むブックマークは「開発」に、「bilibili」や「youtube」は「エンターテイメント」に分類されます。各提案には信頼度スコア（高/中/低）が含まれます。選択的に適用したり、不要なもののチェックを外したりできます。

カテゴリルールは `rules/categories.json` で定義されています。コードに触れることなく、新しいカテゴリを追加したりキーワードを変更したりできます。

また、**設定**ページを開き、「**カスタム分類ルール**」セクションを使用して、UI から直接ルールを追加・編集・削除することもできます。ここで作成されたルールはブラウザのストレージに保存され、`categories.json` のデフォルトルールより優先されます。

### 2. 重複検出
3 種類の重複を見つけます：
- **完全一致** — URL が同一のブックマーク
- **正規化重複** — パスは同じだがクエリパラメータのみ異なるブックマーク（例：同じ文書へのリンクで `timestamp` の値が異なるもの）
- **類似重複** — 同じドメインでタイトルと URL パスが高度に類似したブックマーク（例：「React - 公式ドキュメント」と「React Docs」）

結果は上部にフィルタータブで表示されます。各タブは重複グループを表し、タブをクリックするとそのグループのみが表示されます。一括選択して削除したり、右側の 🗑 ボタンで個別に削除したりできます。削除後、再スキャンせずに UI が即座に更新されます。

確認のために、タブにマウスを重ねると完全なブックマークタイトルと URL が表示されます。

### 3. バックアップとロールバック
スキャンのたびに、ブックマークツリーの全体が自動的にバックアップされます。バックアップはブラウザのローカルストレージに保存され、最新の 10 件が保持され、古いものは自動的に削除されます。分類に問題が生じた場合は、「バックアップ管理」ページを開いて「復元」をクリックするとロールバックできます。

また、バックアップを JSON ファイルとして手動でエクスポートして、簡単に移行や再インストールができるようにすることもできます。

### 4. デュアルインターフェース
ツールバーのアイコンをクリックすると、**フルページインターフェース**（`options.html`）が開きます。左側のサイドバーには、スキャン、分類、重複、バックアップ、設定の 5 つのページへのナビゲーションがあります。外部をクリックしてもページは閉じません。

## インストール

### 方法 1: 開発者モード（テストまたは個人使用向け）

1. リポジトリをクローンします：
   ```bash
   git clone https://github.com/cheechang/BookmarkOrganizer.git
   cd BookmarkOrganizer
   ```

2. `icons/` ディレクトリに、以下の正確なサイズの PNG ファイルが含まれていることを確認してください：
   - `icon16.png` — 16×16
   - `icon48.png` — 48×48
   - `icon128.png` — 128×128

   SVG のみの場合は、オンライン変換ツールを使用して PNG を生成してください。

3. ブラウザの拡張機能管理ページ（Edge の場合は `edge://extensions/`）を開き、右上隅の「開発者モード」を有効にします。

4. 「パッケージ化されていない拡張機能を読み込む」をクリックし、プロジェクトのルートディレクトリを選択します。

5. ツールバーに拡張機能のアイコンが表示されます。クリックして使用を開始します。

### 方法 2: ブラウザ拡張機能ストア

Microsoft Edge Add-ons に提出済み。承認後、「Bookmark Organizer」で検索してインストールできます。

## 使い方

**ステップ 1: スキャン**

拡張機能を開き、「スキャン開始」をクリックします。拡張機能はすべてのブックマークを走査し、未分類数、重複数、分類済み数を分析します。スキャン前に自動的にバックアップが作成されます。

**ステップ 2: 分類**

「分類提案」ページに切り替えると、推奨されるフォルダ構造と各ブックマークの移動先が表示されます。移動しない項目のチェックを外し、「選択した分類を適用」をクリックします。拡張機能は不足しているフォルダを自動的に作成し、ブックマークを移動します。

**ステップ 3: 重複を削除**

「重複検出」ページに切り替えます。上部のタブの列に、各重複グループの名前と数が表示されます。タブをクリックしてそのグループを絞り込みます。拡張機能は**デフォルトで重複をスマートに選択**します：ブックマークバー内のコピーは保持され、外部の重複は自動的にチェックされてワンクリックでクリーンアップできます。手動で選択を調整したり、右側の削除ボタンで個別のブックマークを削除したりすることもできます。

**ステップ 4: バックアップ管理**

「バックアップ管理」ページには、すべての履歴バックアップが一覧表示され、作成時刻とブックマーク数が表示されます。「復元」をクリックしてロールバックしたり、「削除」をクリックしてストレージ領域を解放したりできます。重要なバックアップは定期的にローカル JSON ファイルとしてエクスポートすることをお勧めします。

## カテゴリルールのカスタマイズ

### 方法 1: 設定ページの UI から（推奨）

拡張機能の**設定**ページを開き、「**カスタム分類ルール**」セクションを見つけて「**ルールを追加**」をクリックします。各ルールには以下が必要です：

- **分類名** — 作成されるフォルダ名（例：「仕事関連」）
- **マッチングキーワード** — カンマ区切り。ブックマークのタイトルと照合されます（例：`work, office, meeting`）
- **マッチングドメイン** — カンマ区切り。ブックマークの URL と照合されます（例：`slack.com, notion.so`）

ルールは自動的にブラウザのストレージに保存され、次回のスキャン時に即座に有効になります。いつでも編集または削除できます。

> **注意:** この方法は拡張機能のソースファイルを変更する必要があり、**ソースから拡張機能をビルドしたい開発者**向けです。一般ユーザーは**方法 1**をお使いください。

### 方法 2: `rules/categories.json` を直接編集

JSON ファイルを編集し、以下の形式でエントリを追加します：

```json
{
  "categories": [
    {
      "name": "あなたのカテゴリ",
      "keywords": ["キーワード1", "キーワード2"],
      "domains": ["example.com", "test.org"]
    }
  ]
}
```

変更後、拡張機能管理ページで拡張機能カードの更新ボタンをクリックして拡張機能を再読み込みしてください。

## プロジェクト構造

| ファイル / ディレクトリ | 説明 |
|---|---|
| `manifest.json` | 拡張機能の設定、Manifest V3 |
| `background.js` | Service Worker。インストールイベントとアイコンクリックを処理 |
| `popup.html` / `popup.css` / `popup.js` | ポップアップインターフェース（クイックモード） |
| `options.html` / `options.css` / `options.js` | フルページのスタンドアロンインターフェース |
| `utils.js` | 共通ユーティリティ：分類エンジン、重複検出、バックアップロジック、ブックマーク操作 |
| `rules/categories.json` | デフォルトの分類ルール |
| `icons/` | 拡張機能のアイコン（16px、48px、128px） |
| `docs/` | 開発ドキュメントと変更ログ |
| `PRIVACY_POLICY.md` | プライバシーポリシー（ストア提出に必要） |

核心ロジックは `utils.js` にあります：
- `analyzeBookmarks()` — スキャンして分類提案と重複検出結果を生成
- `detectDuplicates()` — URL とタイトルの類似度に基づいて重複を検出
- `createBackup()` / `restoreBackup()` — バックアップの作成と復元
- `batchMoveBookmarks()` — ブックマークを指定フォルダに一括移動

## 技術的詳細

- **Manifest V3**、完全クライアントサイド、バックエンドサービスなし
- すべてのデータはブラウザの Storage API を使用。ネットワークリクエストなし
- CSP 準拠：インライン `onclick` なし。すべてのイベントは `addEventListener` でバインド
- 類似度アルゴリズムはレーベンシュタイン距離に基づき、タイトルと URL パスの両方を比較し加重平均（タイトル 60%、URL 40%）を使用。閾値は設定で調整可能（デフォルト 80%）
- フォルダ削除時に空でないディレクトリ用に自動的に `removeTree()` を使用し、`remove()` の例外を回避

---

## 既知の制限

- ブックマークが数千件ある場合、スキャンに数秒かかり、UI にプログレスバーが表示されることがあります
- バックアップデータはブラウザのローカルに保存され、拡張機能をアンインストールすると失われます。重要なバックアップはファイルにエクスポートしてください
- カスタムルールの名前が既存のフォルダと同じ場合、ブックマークがそのフォルダに統合される可能性があります
