# 备份管理与持久化功能说明

## 📋 问题背景

### 问题1: 备份存储路径
用户希望看到备份文件的实际存储位置,并能够自定义存储路径。

**技术限制**:
- Chrome扩展使用`chrome.storage.local` API存储数据
- 数据存储在浏览器内部的沙盒数据库中
- **无法直接访问文件系统路径**
- **不支持自定义存储路径**(这是Chrome的安全策略)

### 问题2: 弹窗失焦关闭
用户反馈popup窗口失去焦点就会关闭,导致扫描结果丢失。

**技术原因**:
- 这是Chrome扩展的标准行为
- Popup设计为临时交互界面
- 失去焦点自动关闭是预期行为

## ✅ 解决方案

### 方案1: 备份导出/导入功能

既然无法直接访问存储路径,我们提供**导出/导入**功能作为替代方案:

#### 功能特性

1. **导出备份到文件**
   - 将备份保存为JSON文件
   - 用户可以**选择任意保存位置**
   - 文件名格式: `bookmark-backup-YYYY-MM-DD.json`
   - 支持手动选择保存路径(通过`saveAs: true`)

2. **从文件导入备份**
   - 支持从JSON文件恢复备份
   - 可以导入之前导出的备份
   - 也可以分享备份文件给他人

3. **备份列表管理**
   - 显示所有本地备份
   - 每个备份显示: 时间、书签数量
   - 支持恢复和删除操作

#### 使用方法

**导出备份**:
1. 打开"完整版"页面(options.html)
2. 切换到"备份管理"标签
3. 点击"📤 导出备份到文件"
4. 选择保存位置和文件名
5. 完成!备份已保存到您的电脑

**导入备份**:
1. 点击"📥 从文件导入备份"
2. 选择之前导出的JSON文件
3. 确认导入(会覆盖当前书签)
4. 完成!备份已恢复

### 方案2: 独立完整功能页面

为解决弹窗失焦问题,创建了**完整的独立页面**(options.html):

#### 优势

1. **不会因失焦而关闭**
   - 作为独立标签页打开
   - 可以最小化、切换标签
   - 扫描结果持久保存

2. **更大的工作空间**
   - 侧边栏导航,清晰的功能分区
   - 更大的显示区域
   - 更舒适的操作体验

3. **完整的功能**
   - 扫描分析
   - 分类建议
   - 重复检测
   - 备份管理(含导出/导入)
   - 设置页面

4. **更好的用户体验**
   - 现代化的UI设计
   - 流畅的页面切换
   - 实时统计信息
   - 友好的空状态提示

#### 访问方式

**从Popup打开**:
- 点击Popup底部的"🔗 打开完整版 (推荐)"链接
- 自动在新标签页打开完整功能页面

**直接访问**:
- 右键点击扩展图标
- 选择"选项"或"Options"
- 或直接访问: `chrome-extension://[扩展ID]/options.html`

## 🔧 技术实现

### 1. 导出备份实现

```javascript
async function handleExportBackup() {
  // 获取备份数据
  const result = await chrome.storage.local.get(['bookmarksBackups']);
  const backups = result.bookmarksBackups || [];
  
  // 转换为JSON字符串
  const dataStr = JSON.stringify(backups[0], null, 2);
  const blob = new Blob([dataStr], { type: 'application/json' });
  const url = URL.createObjectURL(blob);
  
  // 触发下载,允许用户选择保存位置
  await chrome.downloads.download({
    url: url,
    filename: `bookmark-backup-${date}.json`,
    saveAs: true  // 关键: 让用户选择位置
  });
}
```

### 2. 导入备份实现

```javascript
async function handleImportBackup(event) {
  const file = event.target.files[0];
  const text = await file.text();
  const backup = JSON.parse(text);
  
  // 验证格式
  if (!backup.data || !backup.timestamp) {
    throw new Error('无效的备份文件格式');
  }
  
  // 恢复书签
  await restoreBookmarks(backup.data);
  
  // 添加到备份列表
  const backups = result.bookmarksBackups || [];
  backups.unshift(backup);
  await chrome.storage.local.set({ bookmarksBackups: backups });
}
```

## 📍 关于存储位置的说明

### Chrome内部存储
```
实际路径(Windows示例):
C:\Users\[用户名]\AppData\Local\Google\Chrome\User Data\Default\Local Extension Settings\[扩展ID]

注意:
- 这是SQLite数据库格式
- 无法直接读取或修改
- 不同操作系统路径不同
- 浏览器更新可能改变位置
```

### 为什么不能自定义路径?
1. **安全沙盒**: Chrome扩展运行在沙盒环境中
2. **跨平台兼容**: 统一API,无需关心具体路径
3. **数据隔离**: 防止恶意扩展访问文件系统
4. **自动同步**: Chrome可以同步extension storage(如果启用)

### 我们的解决方案优势
✅ **灵活性**: 用户可以保存到任何位置  
✅ **可控性**: 完全掌握备份文件  
✅ **可分享**: 可以轻松分享给他人  
✅ **可归档**: 方便长期保存和管理  
✅ **跨设备**: 可以通过云盘同步备份文件
