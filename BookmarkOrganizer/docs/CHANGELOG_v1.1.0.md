## 💡 使用建议

### 推荐使用场景

**Popup模式** (快速模式):
- 快速查看书签状态
- 简单的分类应用
- 临时检查重复项
- 少量书签操作

**完整版** (推荐用于深度整理):
- 大量书签整理 (>100个)
- 需要导出备份文件
- 长时间操作任务
- 多任务并行处理
- 需要持久化工作状态

### 备份最佳实践

1. **定期导出**: 每周或每月导出备份到外部存储
2. **重要操作前**: 批量移动或删除前先备份
3. **多版本保留**: 保留多个历史备份版本
4. **云端同步**: 将备份文件保存到云盘

## ⚠️ 重要说明

### 关于存储位置

**技术限制**:
- Chrome扩展使用`chrome.storage.local` API
- 数据存储在浏览器内部沙盒数据库
- **无法直接访问文件系统路径**
- **不支持自定义存储路径**

**我们的解决方案**:
- ✅ 提供导出/导入功能
- ✅ 用户可以保存到任意位置
- ✅ 支持文件分享和归档
- ✅ 完全掌控备份数据

### 关于弹窗行为

**标准行为**:
- Chrome popup失去焦点会自动关闭
- 这是设计预期,不是bug

**我们的解决方案**:
- ✅ 创建独立完整页面(options.html)
- ✅ 通过"打开完整版"链接访问
- ✅ 不会因失焦而关闭
- ✅ 可以最小化、切换标签

## 📝 已知问题

1. **大型书签库性能**
   - 超过5000个书签时,初次扫描可能较慢
   - 建议分批处理或使用完整版页面
   
2. **浏览器兼容性**
   - 目前仅支持Chrome/Chromium浏览器
   - Firefox版本计划中

3. **存储空间限制**
   - chrome.storage.local有5-10MB限制
   - 建议定期导出备份并清理旧备份
